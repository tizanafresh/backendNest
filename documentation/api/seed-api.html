🌱 API de Seeding

Tizanas Fresh - Documentación de Endpoints de Población de Datos

Módulo para poblar la base de datos con datos de prueba

📋 Descripción General

El módulo de seeding de Tizanas Fresh proporciona endpoints para poblar la base de datos con datos de prueba completos y realistas. Este módulo es esencial para el desarrollo, testing y demostración del sistema, creando un entorno de datos consistente y funcional.

🎯 Funcionalidades Principales:
  • Población Completa: Crea todos los datos necesarios para el funcionamiento del sistema
  • Datos Realistas: Información coherente y realista para testing
  • Limpieza Automática: Elimina datos existentes antes de crear nuevos
  • Estado de Proceso: Monitoreo del estado de la operación de seeding
  • Relaciones Inteligentes: Mantiene referencias correctas entre entidades
⚠️ Advertencia Importante:
  • Este módulo está diseñado SOLO para desarrollo y testing
  • NO usar en producción bajo ninguna circunstancia
  • Elimina TODOS los datos existentes antes de crear nuevos
  • Requiere permisos de administrador para funcionar correctamente
📊 Datos que se Crean:
  • 5 Categorías: Frutas, Detox, Energética, Relajante, Especiales
  • 5 Productos: Tizanas con ingredientes y información nutricional
  • 4 Usuarios: Admin, María, Carlos, Ana con diferentes niveles
  • 3 Cupones: WELCOME10, FREESHIP, LOYALTY20
  • 2 Pedidos: Con items, direcciones y estados
  • 4 Notificaciones: Diferentes tipos y estados
  • 4 Registros de Fidelización: Historial de puntos
  • 3 Tokens de Dispositivos: Para notificaciones push
📝 Nota: Todos los endpoints de seeding están disponibles en el prefijo /seed y el servidor corre en http://localhost:5001

🌱 Endpoints de Seeding

POST Ejecutar Seeding Completo

/seed
📖 Descripción General:

Este endpoint ejecuta el proceso completo de seeding, eliminando todos los datos existentes y creando nuevos datos de prueba. El proceso es secuencial y crea las entidades en el orden correcto para mantener las referencias entre ellas.

🎯 Casos de Uso:
  • Configuración inicial: Poblar base de datos vacía al iniciar desarrollo
  • Reset de datos: Limpiar y recrear datos para testing limpio
  • Demostración: Preparar datos para presentaciones o demos
  • Testing de integración: Crear entorno consistente para pruebas
  • Desarrollo de frontend: Proporcionar datos realistas para desarrollo de UI

📥 Request Body

📝 Nota: Este endpoint no requiere body. Solo envía una petición POST vacía.

📤 Response (200 OK)

Seeding completado exitosamente:
{ "message": "Seeding completado exitosamente", "summary": { "categories": 5, "users": 4, "products": 5, "coupons": 3, "orders": 2, "notifications": 4, "loyaltyHistory": 4, "deviceTokens": 3 }, "details": { "categories": ["Frutas", "Detox", "Energética", "Relajante", "Especiales"], "users": ["Admin Tizanas", "María González", "Carlos Rodríguez", "Ana Martínez"], "products": ["Tizana de Frutos Rojos", "Tizana Tropical", "Tizana Detox Verde", "Tizana Energética", "Tizana Relajante"] }, "timestamp": "2024-01-15T10:30:00.000Z", "duration": "2.5s" }

❌ Response (500 Internal Server Error)

Error durante el seeding:
{ "statusCode": 500, "message": "Error durante el seeding: No se pudo conectar a la base de datos", "error": "Internal Server Error" }

🧪 Ejemplo de Uso con cURL

curl -X POST http://localhost:5001/seed \ -H "Content-Type: application/json"

🧪 Ejemplo de Uso con JavaScript

const response = await fetch('http://localhost:5001/seed', { method: 'POST', headers: { 'Content-Type': 'application/json' } }); const result = await response.json(); console.log('Seeding completado:', result.message); console.log('Datos creados:', result.summary);

GET Verificar Estado del Seeding

/seed/status
📖 Descripción General:

Este endpoint verifica el estado actual de la base de datos, mostrando cuántos registros existen en cada colección. Es útil para verificar si el seeding se ejecutó correctamente o para diagnosticar problemas.

🎯 Casos de Uso:
  • Verificación post-seeding: Confirmar que todos los datos se crearon correctamente
  • Diagnóstico: Verificar el estado actual de la base de datos
  • Monitoreo: Revisar qué datos están disponibles
  • Debugging: Identificar problemas en el proceso de seeding
  • Validación: Asegurar que las relaciones entre entidades son correctas

📤 Response (200 OK)

Estado actual de la base de datos:
{ "status": "populated", "collections": { "categories": { "count": 5, "names": ["Frutas", "Detox", "Energética", "Relajante", "Especiales"] }, "users": { "count": 4, "emails": ["admin@tizanafresh.com", "maria@example.com", "carlos@example.com", "ana@example.com"] }, "products": { "count": 5, "names": ["Tizana de Frutos Rojos", "Tizana Tropical", "Tizana Detox Verde", "Tizana Energética", "Tizana Relajante"] }, "coupons": { "count": 3, "codes": ["WELCOME10", "FREESHIP", "LOYALTY20"] }, "orders": { "count": 2, "statuses": ["DELIVERED", "PREPARING"] }, "notifications": { "count": 4, "types": ["ORDER", "PROMOTION", "SYSTEM"] }, "loyaltyHistory": { "count": 4, "types": ["EARNED", "BONUS", "PROMOTION"] }, "deviceTokens": { "count": 3, "platforms": ["ANDROID", "IOS"] } }, "lastSeeding": "2024-01-15T10:30:00.000Z", "database": "tizanafresh" }

const response = await fetch('http://localhost:5001/seed/status'); const status = await response.json(); if (status.status === 'populated') { console.log('✅ Base de datos poblada correctamente'); console.log('Usuarios:', status.collections.users.count); console.log('Productos:', status.collections.products.count); } else { console.log('❌ Base de datos vacía, ejecutar seeding'); }

🔧 Proceso de Seeding

📋 Orden de Creación:
  1. Categorías: Se crean primero para referenciar en productos
  2. Usuarios: Se crean con contraseñas hasheadas
  3. Productos: Se asocian a categorías existentes
  4. Cupones: Algunos se asocian a usuarios específicos
  5. Pedidos: Se crean con items de productos existentes
  6. Notificaciones: Se asocian a usuarios específicos
  7. Historial de Fidelización: Se vincula a usuarios y pedidos
  8. Tokens de Dispositivos: Se asocian a usuarios
⚡ Características del Proceso:
  • Transaccional: Si falla un paso, se revierte todo
  • Limpieza previa: Elimina todos los datos existentes
  • Logging detallado: Muestra progreso en consola
  • Validación: Verifica que las relaciones sean correctas
  • Performance: Optimizado para crear datos rápidamente

🚀 Flujo de Uso Completo

1. Verificar Estado Inicial:
GET /seed/status Response: { "status": "empty", "collections": { ... } }
2. Ejecutar Seeding:
POST /seed Response: { "message": "Seeding completado exitosamente", "summary": { "categories": 5, "users": 4, "products": 5, ... } }
3. Verificar Resultado:
GET /seed/status Response: { "status": "populated", "collections": { "users": { "count": 4, "emails": ["admin@tizanafresh.com", ...] }, ... } }
4. Probar Autenticación:
POST /auth/login { "email": "admin@tizanafresh.com", "password": "password123" } Response: { "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "user": { ... } }
