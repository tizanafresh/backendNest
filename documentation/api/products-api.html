🥤 API de Productos

Tizanas Fresh - Documentación de Endpoints de Gestión de Productos

Módulo para gestionar el catálogo de tizanas y bebidas

📋 Descripción General

El módulo de productos de Tizanas Fresh proporciona endpoints para gestionar el catálogo completo de tizanas y bebidas. Este módulo permite crear, consultar, actualizar y eliminar productos, así como gestionar categorías y obtener información nutricional detallada.

🎯 Funcionalidades Principales:
  • Gestión de Productos: CRUD completo para tizanas y bebidas
  • Categorización: Organización por tipos (Frutas, Detox, Energética, etc.)
  • Información Nutricional: Detalles de calorías, proteínas, carbohidratos
  • Gestión de Inventario: Control de disponibilidad de productos
  • Búsqueda y Filtrado: Consultas avanzadas por categoría, precio, etc.
  • Imágenes de Productos: URLs para mostrar imágenes de las tizanas
📊 Tipos de Productos Disponibles:
  • Tizanas de Frutas: Combinaciones refrescantes de frutas naturales
  • Tizanas Detox: Bebidas purificantes con ingredientes detox
  • Tizanas Energéticas: Bebidas con ingredientes energizantes
  • Tizanas Relajantes: Bebidas calmantes con hierbas relajantes
  • Tizanas Especiales: Combinaciones únicas y de temporada
📝 Nota: Todos los endpoints de productos están disponibles en el prefijo /products y el servidor corre en http://localhost:5001

🥤 Endpoints de Productos

GET Obtener Todos los Productos

/products
📖 Descripción General:

Este endpoint devuelve una lista paginada de todos los productos disponibles en el catálogo. Permite filtrado por categoría, disponibilidad y ordenamiento por precio o nombre. Es ideal para mostrar el catálogo completo en la aplicación.

🎯 Casos de Uso:
  • Catálogo principal: Mostrar todos los productos en la página principal
  • Filtrado por categoría: Mostrar solo tizanas de frutas o detox
  • Búsqueda de productos: Filtrar por disponibilidad o precio
  • Ordenamiento: Mostrar productos por precio ascendente/descendente
  • Paginación: Cargar productos en lotes para mejor performance

📥 Query Parameters

Parámetro Tipo Requerido Descripción Ejemplo
page number No Número de página (default: 1) 1
limit number No Productos por página (default: 10) 20
category string No Filtrar por ID de categoría 507f1f77bcf86cd799439011
available boolean No Filtrar por disponibilidad true
sort string No Ordenar por campo (price, name) price
order string No Orden (asc, desc) asc

📤 Response (200 OK)

Lista de productos paginada:
{ "products": [ { "id": "507f1f77bcf86cd799439011", "name": "Tizana de Frutos Rojos", "description": "Refrescante mezcla de fresas, frambuesas y moras con hierbas naturales", "price": 8.50, "category": { "id": "507f1f77bcf86cd799439012", "name": "Frutas" }, "image": "https://example.com/tizana-frutos-rojos.jpg", "available": true, "ingredients": ["fresas", "frambuesas", "moras", "hierbabuena", "limón"], "nutritionalInfo": { "calories": 45, "protein": 1.2, "carbs": 10.5, "fat": 0.3, "fiber": 3.2 }, "createdAt": "2024-01-01T00:00:00.000Z", "updatedAt": "2024-01-15T10:30:00.000Z" } ], "pagination": { "page": 1, "limit": 10, "total": 5, "pages": 1 }, "filters": { "category": null, "available": null, "sort": "name", "order": "asc" } }

🧪 Ejemplo de Uso con cURL

# Obtener todos los productos curl -X GET "http://localhost:5001/products" # Filtrar por categoría curl -X GET "http://localhost:5001/products?category=507f1f77bcf86cd799439012" # Ordenar por precio descendente curl -X GET "http://localhost:5001/products?sort=price&order=desc" # Solo productos disponibles curl -X GET "http://localhost:5001/products?available=true"

🧪 Ejemplo de Uso con JavaScript

// Obtener todos los productos const response = await fetch('http://localhost:5001/products'); const data = await response.json(); console.log('Productos:', data.products); console.log('Total:', data.pagination.total); // Filtrar por categoría const categoryResponse = await fetch('http://localhost:5001/products?category=507f1f77bcf86cd799439012'); const categoryData = await categoryResponse.json(); console.log('Productos de frutas:', categoryData.products);

GET Obtener Producto por ID

/products/:id
📖 Descripción General:

Este endpoint devuelve la información detallada de un producto específico por su ID. Incluye toda la información del producto, su categoría, ingredientes, información nutricional y estado de disponibilidad.

🎯 Casos de Uso:
  • Página de producto: Mostrar detalles completos de una tizana
  • Carrito de compras: Obtener información para agregar al carrito
  • Comparación de productos: Obtener datos para comparar tizanas
  • Información nutricional: Mostrar detalles nutricionales
  • Gestión de inventario: Verificar disponibilidad de producto

📥 Path Parameters

Parámetro Tipo Requerido Descripción Ejemplo
id string Sí ID único del producto 507f1f77bcf86cd799439011

POST Crear Nuevo Producto

/products
📖 Descripción General:

Este endpoint permite crear un nuevo producto en el catálogo. Requiere autenticación de administrador y valida todos los campos obligatorios. El producto se crea con información completa incluyendo ingredientes, información nutricional y categoría.

🎯 Casos de Uso:
  • Nuevo producto: Agregar una nueva tizana al catálogo
  • Producto de temporada: Crear tizanas especiales de temporada
  • Expansión de menú: Agregar nuevas variedades de bebidas
  • Gestación de inventario: Preparar productos antes del lanzamiento
🔒 Autenticación: Requiere token JWT de administrador

📥 Request Body

Campo Tipo Requerido Descripción Ejemplo
name string Sí Nombre del producto Tizana de Mango y Piña
description string Sí Descripción detallada Exótica combinación de mango dulce y piña refrescante
price number Sí Precio en dólares 9.50
category string Sí ID de la categoría 507f1f77bcf86cd799439012
image string No URL de la imagen https://example.com/tizana-mango-pina.jpg
available boolean No Disponibilidad (default: true) true
ingredients string[] No Lista de ingredientes ["mango", "piña", "jengibre", "lima"]
nutritionalInfo object No Información nutricional {"calories": 52, "protein": 0.8}

📤 Response (201 Created)

Producto creado exitosamente:
{ "id": "507f1f77bcf86cd799439013", "name": "Tizana de Mango y Piña", "description": "Exótica combinación de mango dulce y piña refrescante", "price": 9.50, "category": { "id": "507f1f77bcf86cd799439012", "name": "Frutas" }, "image": "https://example.com/tizana-mango-pina.jpg", "available": true, "ingredients": ["mango", "piña", "jengibre", "lima"], "nutritionalInfo": { "calories": 52, "protein": 0.8, "carbs": 12.1, "fat": 0.5, "fiber": 2.8 }, "createdAt": "2024-01-15T10:30:00.000Z", "updatedAt": "2024-01-15T10:30:00.000Z" }

❌ Response (400 Bad Request)

Datos inválidos:
{ "statusCode": 400, "message": ["name should not be empty", "price must be a positive number"], "error": "Bad Request" }

❌ Response (401 Unauthorized)

Sin autenticación:
{ "statusCode": 401, "message": "Unauthorized", "error": "Unauthorized" }

🧪 Ejemplo de Uso con cURL

curl -X POST "http://localhost:5001/products" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \ -d '{ "name": "Tizana de Mango y Piña", "description": "Exótica combinación de mango dulce y piña refrescante", "price": 9.50, "category": "507f1f77bcf86cd799439012", "ingredients": ["mango", "piña", "jengibre", "lima"], "nutritionalInfo": { "calories": 52, "protein": 0.8, "carbs": 12.1, "fat": 0.5 } }'

🧪 Ejemplo de Uso con JavaScript

const token = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'; const newProduct = { name: 'Tizana de Mango y Piña', description: 'Exótica combinación de mango dulce y piña refrescante', price: 9.50, category: '507f1f77bcf86cd799439012', ingredients: ['mango', 'piña', 'jengibre', 'lima'], nutritionalInfo: { calories: 52, protein: 0.8, carbs: 12.1, fat: 0.5 } }; const response = await fetch('http://localhost:5001/products', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` }, body: JSON.stringify(newProduct) }); const createdProduct = await response.json(); console.log('Producto creado:', createdProduct.name);

PUT Actualizar Producto

/products/:id
📖 Descripción General:

Este endpoint permite actualizar la información de un producto existente. Solo se actualizan los campos proporcionados, manteniendo los valores existentes para los campos no incluidos. Requiere autenticación de administrador.

🎯 Casos de Uso:
  • Actualizar precio: Modificar precios de productos
  • Cambiar disponibilidad: Marcar productos como no disponibles
  • Actualizar descripción: Mejorar descripciones de productos
  • Agregar ingredientes: Actualizar lista de ingredientes
  • Cambiar imagen: Actualizar foto del producto
🔒 Autenticación: Requiere token JWT de administrador

📥 Path Parameters

Parámetro Tipo Requerido Descripción Ejemplo
id string Sí ID único del producto 507f1f77bcf86cd799439011

📥 Request Body

📝 Nota: Todos los campos son opcionales. Solo se actualizan los campos proporcionados.

📤 Response (200 OK)

Producto actualizado:
{ "id": "507f1f77bcf86cd799439011", "name": "Tizana de Frutos Rojos Premium", "description": "Refrescante mezcla de fresas, frambuesas y moras con hierbas naturales", "price": 9.50, "category": { "id": "507f1f77bcf86cd799439012", "name": "Frutas" }, "image": "https://example.com/tizana-frutos-rojos-premium.jpg", "available": true, "ingredients": ["fresas", "frambuesas", "moras", "hierbabuena", "limón", "jengibre"], "nutritionalInfo": { "calories": 45, "protein": 1.2, "carbs": 10.5, "fat": 0.3, "fiber": 3.2 }, "updatedAt": "2024-01-15T11:00:00.000Z" }

🧪 Ejemplo de Uso con cURL

curl -X PUT "http://localhost:5001/products/507f1f77bcf86cd799439011" \ -H "Content-Type: application/json" \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \ -d '{ "price": 9.50, "name": "Tizana de Frutos Rojos Premium" }'

DELETE Eliminar Producto

/products/:id
📖 Descripción General:

Este endpoint elimina permanentemente un producto del catálogo. La eliminación es irreversible y también elimina todas las referencias al producto en pedidos y otros módulos. Requiere autenticación de administrador.

🎯 Casos de Uso:
  • Producto discontinuado: Eliminar productos que ya no se venden
  • Limpieza de catálogo: Remover productos obsoletos
  • Corrección de errores: Eliminar productos creados por error
  • Gestión de inventario: Remover productos sin stock
⚠️ Advertencia: Esta operación es irreversible y elimina permanentemente el producto y todas sus referencias.
🔒 Autenticación: Requiere token JWT de administrador

📤 Response (200 OK)

Producto eliminado:
{ "message": "Producto eliminado exitosamente", "deletedProduct": { "id": "507f1f77bcf86cd799439011", "name": "Tizana de Frutos Rojos" } }

🧪 Ejemplo de Uso con cURL

curl -X DELETE "http://localhost:5001/products/507f1f77bcf86cd799439011" \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

🚀 Flujo de Gestión de Productos

1. Crear Nuevo Producto:
POST /products { "name": "Nueva Tizana", "description": "Descripción del producto", "price": 8.50, "category": "507f1f77bcf86cd799439012" } Response: 201 Created { "id": "507f1f77bcf86cd799439013", "name": "Nueva Tizana", ... }
2. Obtener Producto:
GET /products/507f1f77bcf86cd799439013 Response: 200 OK { "id": "507f1f77bcf86cd799439013", "name": "Nueva Tizana", ... }
3. Actualizar Producto:
PUT /products/507f1f77bcf86cd799439013 { "price": 9.00 } Response: 200 OK { "id": "507f1f77bcf86cd799439013", "price": 9.00, ... }
4. Eliminar Producto:
DELETE /products/507f1f77bcf86cd799439013 Response: 200 OK { "message": "Producto eliminado exitosamente" }
